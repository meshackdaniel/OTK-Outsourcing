OaaS (Outsourcing as a Service)
End-to-End Operational, Revenue & Marketing Workflow
DOCUMENT PURPOSE
This document defines the complete operational, financial, and marketing architecture of the OaaS platform.
OaaS operates as a multi-revenue managed workforce & software platform, not merely a job board.
PLATFORM POSITIONING
OaaS is a: Workforce infrastructure platform providing hiring, managed outsourcing, and recruitment software solutions.
CORE OPERATING PRINCIPLES
Wallet-First Control
•	No deployment without funded wallet
•	No service activation without payment
•	No payout without approved logs
Revenue Before Disbursement
•	Platform earnings are retained before any payout.

Compliance by Design
•	PAYE, VAT, and WHT embedded at transaction stages.

Modular Revenue Architecture
•	Each module operates independently but feeds into one financial control system.

OaaS EARNINGS & COMMISSION MODULE
JOB BOARD & TALENT MARKETPLACE
Revenue Models
Success Fee (Managed Agency Model)
Platform earns only when hire is successful.
Typical range:
15% – 30% of annual salary
Example:
Personnel annual salary: ₦10,000,000
20% fee = ₦2,000,000

Retained / Executive Search
Milestone-based fee structure:
•	Upfront initiation fee
•	Shortlist milestone
•	Interview milestone
•	Final hire payment
Revenue is milestone-based, not salary-based.
SYSTEM LOGIC FLOW
Step 1 – Contract Creation
Admin defines:
•	Total contract value
•	Milestone breakdown %
•	Deliverable criteria
•	Timeline
•	Approval workflow
Step 2 – Wallet Funding
Client must:
•	Fund full contract value OR
•	Fund first milestone before activation
System rule:
Milestone cannot activate without reserved funds.
Step 3 – Fund Reservation
For each milestone:
•	Funds moved from Client Wallet
•	Into Milestone Reserved Wallet
Reserved funds cannot be withdrawn.
Step 4 – Work Execution
Assigned team/vendor/talent works toward milestone deliverable.
Step 5 – Milestone Completion Submission
Responsible party submits:
•	Deliverable documentation
•	Evidence
•	Confirmation report
Step 6 – Client Review Window
System grants 3–7 day review window.
If no dispute → auto-approval triggers.
Step 7 – Financial Execution
Upon approval:
1.	VAT calculated (if applicable)
2.	Platform margin retained
3.	WHT calculated (if applicable)
4.	Net payout credited to vendor/talent wallet
5.	Ledger updated
6.	Tax liability recorded
FINANCIAL LOGIC FORMULA
For each milestone:
Milestone Value (MV)
Example:
MV = ₦1,500,000
Platform Margin (PM)
Example: 20%
PM = ₦300,000
Service Value = ₦1,200,000
VAT (7.5% on service component only)
VAT = 7.5% × ₦300,000 (if VAT only applies to service fee)
WHT (5%–10% depending on service classification)
WHT deducted from vendor payout.
Final Vendor Payout:
MV
– Platform Margin
– WHT
= Net Vendor Credit

TAX LOGIC PER MILESTONE
PAYE
Applies only if milestone relates to employment salary component.
VAT
Applied to service portion of milestone.
WHT
Applies when:
•	Vendor is external service provider
•	Corporate B2B transaction
System automatically:
•	Generates WHT credit note
•	Updates WHT ledger

ESCROW PROTECTION LOGIC
Milestone funds behave like escrow:
Client funds → Reserved Wallet →
Released only upon milestone approval.
Dispute scenario:
•	Funds frozen
•	Admin mediation triggered
•	Escalation protocol activated
If dispute unresolved:
•	Governing SLA clause executed
•	Partial release possible
DISPUTE RESOLUTION WORKFLOW
1.	Client raises dispute within review window.
2.	System freezes milestone.
3.	Evidence upload required.
4.	Admin review within 5 business days.
5.	Resolution options:
o	Full release
o	Partial release
o	Refund to client wallet
o	Rework required
No funds move without admin authorization.

CANCELLATION LOGIC
If client cancels after milestone 1 completion:
Completed milestones = payable.
Uncompleted milestones = refunded from reserved wallet.
If cancellation occurs mid-milestone:
Admin calculates:
•	% completion
•	Fair value
•	Partial payout

Personnel Work Log Workflow 
The Personnel Work Log system ensures that outsourced staff work records are verified before payroll and client billing are processed.
Log Submission
Personnel submit daily or weekly work logs including:
•	Hours worked
•	Tasks performed
•	Overtime (if applicable)
•	Supporting evidence (where required)
Late or incomplete entries are flagged automatically.
System Validation
The platform checks for:
•	Missing information
•	Duplicate entries
•	Excessive hours
•	Overtime beyond policy limits
Errors must be corrected before review.
Client Approval
The assigned client supervisor:
•	Approves, rejects, or requests corrections
•	Must act within a defined review window
Unreviewed logs escalate to Admin.
Admin Oversight
Admin handles:
•	Disputes
•	Overtime exceptions
•	Suspicious entries
•	Final approval where necessary
No payroll proceeds without proper approval.
Payroll & Billing Integration
Approved logs determine:
•	Salary calculations (for hourly or hybrid models)
•	Overtime payments
•	Absence deductions
•	Client invoices (for time-based outsourcing)
Payroll is executed only after approved logs are aggregated.
Compliance & Controls
The system maintains:
•	Time-stamped audit trail
•	Approval records
•	Edit history
•	Escalation process
This ensures transparency, reduces disputes, and protects payroll accuracy.

On-Demand Recruiter Rental
(Hourly or Project-Based Engagement Model)
This module allows companies to “rent” a professional recruiter through the OaaS platform without committing to full outsourcing or permanent placement contracts.

SERVICE POSITIONING
On-Demand Recruiter Rental is a flexible hiring support model where:
•	Companies hire a recruiter temporarily.
•	Billing is hourly OR project-based.
•	OaaS manages scheduling, billing, wallet control, and margin retention.
•	Recruiter remains contractor under OaaS governance.
This is NOT full employer-of-record.
It is a time-bound professional service engagement.

ENGAGEMENT MODELS
Hourly Model
Company selects:
•	Recruiter level (Junior / Mid / Senior / Executive)
•	Number of hours required
•	Duration window (e.g., 2 weeks)
Admin sets base hourly rate.
Example:
Junior – ₦10,000/hr
Mid – ₦20,000/hr
Senior – ₦35,000/hr
Executive – ₦60,000/hr
Minimum billable hours may apply (e.g., 10 hours).

Project-Based Model
Company defines hiring project:
•	Example: “Hire 10 Sales Reps within 30 days”
Admin or system calculates:
•	Estimated effort hours
•	Fixed project fee
Example:
₦800,000 project fee (inclusive of 40 estimated hours)
Scope defined in SLA.

OPERATIONAL WORKFLOW
Step 1 – Request Submission
Company submits recruiter request with:
•	Role type
•	Volume
•	Timeline
•	Budget preference (hourly/project)
Step 2 – Pricing Calculation
System calculates:
•	Base fee
•	VAT
•	Estimated total
•	Platform margin
Step 3 – Wallet Funding
Company must fund:
•	Estimated hours (hourly model)
OR
•	Full project fee (project model)
Funds move to Reserved Recruiter Wallet.
Step 4 – Recruiter Assignment
Admin assigns recruiter OR
Company selects from approved recruiter pool.
Step 5 – Work Execution
For Hourly Model:
•	Recruiter logs time via system timesheet.
•	Client confirms weekly timesheet.
For Project Model:
•	Deliverables tracked by milestone (shortlist, interviews, hires).
Step 6 – Approval Trigger
Hourly:
•	Weekly approval window.
•	Auto-approval after review period.
Project:
•	Milestone approval.
Step 7 – Financial Execution
Upon approval:
1.	Platform margin retained.
2.	VAT calculated.
3.	WHT calculated (if applicable).
4.	Net recruiter payout credited.
5.	Ledger updated.

OaaS AFFILIATE COMMISSION WORKFLOW

OBJECTIVE
The Affiliate Commission System is designed to:
•	Incentivize acquisition of Clients and Talents
•	Reward revenue-generating actions only
•	Protect platform margins
•	Prevent abuse and fake registrations
•	Maintain tax and audit compliance
Affiliate commissions are triggered only on monetized events.

AFFILIATE TYPES
OaaS recognizes two primary affiliate referral categories:
Client Referral Affiliate
Refers:
•	Companies
•	Corporate HR teams
•	Enterprise partners
Talent Referral Affiliate
Refers:
•	Skilled professionals
•	Semi-skilled workforce
•	Specialized experts
•	Freelancers
Each referral type has different commission triggers.

AFFILIATE REGISTRATION WORKFLOW
1.	Affiliate registers.
2.	KYC verification completed.
3.	Affiliate Wallet auto-created.
4.	Unique referral ID & tracking link generated.
5.	Commission structure assigned (admin-configurable).
System rule:
No commission is payable without verified KYC.

CLIENT REFERRAL COMMISSION LOGIC
Commission Trigger Events (Client-Side)
Commission is triggered when the referred client:
✔ Activates a paid job listing
✔ Purchases featured job slot
✔ Subscribes to OaaS plan
✔ Pays milestone-based recruitment fee
✔ Funds managed outsourcing payroll
✔ Purchases SaaS tools
✔ Buys event sponsorship
✔ Purchases lead generation services
Commission is NOT triggered when:
✖ Client registers only
✖ Client posts draft jobs
✖ Client does not fund wallet
✖ Client negotiates but does not pay

Commission Calculation Logic (Client Revenue)
Example Model:
Affiliate Commission = 5% – 15% of Platform Margin
(Not total contract value)
Example:
Client pays ₦1,000,000
Platform margin = ₦250,000
Affiliate rate = 10% of margin
Commission = ₦25,000
This protects platform profitability.

Milestone-Based Contracts
If client pays via milestones:
Commission triggers per paid milestone.
Example:
Milestone 1 paid → commission credited
Milestone 2 unpaid → no commission yet
This ensures revenue-backed payout.

TALENT REFERRAL COMMISSION LOGIC
Talent referral is more sensitive because revenue depends on deployment.
There are two models:

MODEL A – Deployment-Based Commission (Primary Model)
Commission triggers ONLY when:
✔ Talent is deployed
✔ Client pays first invoice
✔ Platform retains margin
Commission calculated as:
Percentage of first-month margin only.
Example:
Talent deployed
Client pays ₦500,000 monthly
Platform margin = ₦100,000
Affiliate rate = 20% of first month margin
Commission = ₦20,000
One-time payout only (unless subscription-based contract).

MODEL B – Activation-Based Commission (Optional)
Small incentive if:
Talent completes profile + passes assessment.
Example:
₦2,000 fixed activation bonus.
BUT this requires:
•	Anti-fraud controls
•	Device/IP tracking
•	Unique ID verification
Recommended to cap monthly activation bonuses.

TALENT NOT DEPLOYED SCENARIO
If talent:
✔ Registers
✔ Completes profile
✖ Not deployed
Then:
No deployment-based commission paid.
Optional:
Admin may set:
“Deployment Window”
Example:
If talent not deployed within 180 days → referral expires.
This prevents indefinite commission liability.

SUBSCRIPTION-BASED CLIENT COMMISSION
If client subscribes monthly:
Option A – One-Time Commission
Commission paid only on first subscription month.
Option B – Recurring Commission (Limited Duration)
Commission paid for first 3–6 months only.
Example:
Client pays ₦200,000/month
Margin = ₦80,000
Affiliate rate = 10%
Commission per month = ₦8,000
Valid for 3 months.
Admin-configurable.

WALLET FLOW FOR COMMISSION
Revenue Event Occurs →
Platform Margin Calculated →
Commission Calculated →
WHT Deducted (if applicable) →
Net Commission Credited to Affiliate Wallet →
Withdrawal allowed after holding period.

7–14 days to allow dispute buffer.

VAT
Not applicable to affiliate payout.
WHT
